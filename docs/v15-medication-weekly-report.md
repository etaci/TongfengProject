# V15 补药提醒与依从复盘周报

## 本轮目标

V15 在 V14「每日用药打卡」的基础上，继续把“能每天用”推进到“能每周复盘、能提前补药”。

本轮新增两条能力：

- 补药提醒：在用药计划里维护剩余天数和提醒阈值后，系统会把临近补药的药物写入提醒流。
- 用药依从周报：基于最近 7 天的计划剂次和打卡结果，输出周依从率、连续完成天数、每日拆分、重点药物和补药风险。

## 用药计划扩展

`MedicationItem` 新增两个可选字段：

- `remainingDays`：预计还剩几天药量
- `refillThresholdDays`：提前几天开始提醒补药

前端输入格式：

```text
药名 | 剂量 | 频次 | 备注 | 剩余天数 | 提前提醒阈值
```

示例：

```text
别嘌醇 | 100mg | 每日一次 | 早餐后 | 5 | 3
秋水仙碱 | 0.5mg | 每日两次 | 短期预防 | 2 | 3
```

后两个字段可留空。

## 新增接口

### 1. 用药依从周报

- `GET /api/v1/medications/weekly-report?days=7`

返回内容包括：

- 周计划剂次 / 已服用 / 漏服 / 跳过 / 未说明
- 周依从率
- 当前连续完成天数
- 窗口内最长连续完成天数
- 每日拆分
- 重点药物
- 补药提醒
- 复盘亮点与下一步建议

### 2. 提醒流中的补药提醒

无需新接口，走现有：

- `GET /api/v1/reminders`

当药物 `remainingDays <= refillThresholdDays` 时，会出现：

- `type = MEDICATION_REFILL`

## 安全边界

- 补药提醒只做流程提示，不代替正式开药、处方审核和医保规则判断。
- 周报只根据当前计划和打卡结果生成，不代表医生对依从性的最终判断。
- 如果药物方案已停用或已调整，请先更新用药计划，再参考提醒和周报。
