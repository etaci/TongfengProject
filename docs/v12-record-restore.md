# V12 记录恢复与审计回滚

V12 在 V11 审计能力基础上继续补齐了“可恢复”闭环。

## 新增能力

1. 删除后恢复记录
2. 基于历史审计回滚到更早版本
3. 恢复动作继续写入新的审计记录

## 新增接口

### 恢复记录

- 方法：`POST /api/v1/records/restore`
- Query：
  - `type`：记录类型
  - `recordId`：记录 ID
  - `auditId`：目标审计 ID

请求体：

```json
{
  "changeReason": "undo delete"
}
```

## 恢复规则

### 1. 恢复删除记录

当 `auditId` 对应的是一次 `DELETE` 审计时：

- 系统读取该次删除前的快照
- 重新创建原记录
- 返回恢复后的最新详情

### 2. 回滚更新记录

当 `auditId` 对应的是一次 `UPDATE` 审计时：

- 系统读取该次更新前的快照
- 将当前记录回滚到旧版本
- 返回回滚后的详情

## 审计动作

恢复或回滚成功后，会新增一条 `RESTORE` 审计，便于后续继续追溯：

- `action = RESTORE`
- `changeReason` 为本次恢复原因
- `fields` 记录恢复前后差异

## 当前记录中心闭环

统一记录中心现已支持：

1. 聚合分页查询
2. 详情查询
3. 记录更正
4. 记录删除
5. 审计查询
6. 删除后恢复
7. 历史版本回滚
