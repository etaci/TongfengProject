# V17 家属代办确认闭环

## 本轮目标

V17 把家属协同从“能看、能提醒”推进到“能协助完成具体事情”。

本轮新增能力：

- 患者可向具备 `TASK` 权限的家属创建协同代办
- 家属可查看自己待处理的代办列表
- 家属处理后可直接确认完成，并回填处理说明
- 患者可回看代办状态、完成时间与家属反馈

## 新增接口

### 1. 查看家属代办看板

- `GET /api/v1/family/tasks`

返回：

- `asPatient`：我作为患者发起的代办
- `asCaregiver`：我作为家属待处理或已处理的代办

### 2. 患者创建家属代办

- `POST /api/v1/family/members/{bindingCode}/tasks`

请求体：

```json
{
  "title": "周一上午帮我确认补药",
  "description": "如果剩余药量不足 2 天，请提醒我及时去药房。",
  "dueAt": "2026-05-04T09:00:00Z"
}
```

约束：

- 只有患者本人可发起
- 绑定关系必须为 `ACTIVE`
- 家属权限必须为 `TASK`

### 3. 家属确认代办完成

- `POST /api/v1/family/tasks/{taskCode}/complete`

请求体：

```json
{
  "completionNote": "已和药房确认库存，明早可以处理补药。"
}
```

约束：

- 只有被指派的家属可确认完成
- 绑定关系仍需有效
- 当前任务必须仍是 `OPEN`

## 前端变化

- 家庭协同页新增“发起家属代办”
- 患者可选择具备 `TASK` 权限的家属并下发任务
- 页面新增“家属代办看板”，同时展示：
  - 我发起的代办
  - 我待确认的代办
- 家属可在页面内直接填写处理反馈并确认完成

## 安全边界

- 没有 `TASK` 权限的家属不能接收或完成协同代办
- 家属代办不替代医疗建议，仅用于家庭照护协作
- 解绑后家属不再能继续处理新的协同代办
